## 1. Registration & Anti-Spam Gate

- People must register before joining sessions
- Basic screening: intentionality, agreements, consent to recording
- Ideally auto-approve or auto-email room links to avoid manual bottlenecks
- Ability to prevent last-minute chaos if 30+ people register
- Ability to manually block or remove disruptive users

## 2. Live Room Experience (Non-Zoom)

- Open-source or privacy-respecting platform
- Video on/off, audio participation, chat
- Ability to:
  - Mute all on entry
  - Control who can unmute
  - Spotlight speakers
  - Lock the room after session starts
  - Remove disruptive users
- Recording capability (for most sessions, but not grief circles)

## 3. Speaker / Educator Intake

Separate form where people can propose:
- Topics
- Which day (Mon/Wed/Fri)
- Short bio
- Links or materials

Submissions should:
- Go into a backend database
- Be reviewed before publishing
- Appear publicly only after approval

Public view should show:
- Upcoming proposed sessions
- Who else is already on the docket
- People must not be able to edit others' submissions

## 4. Discoverability & Publishing

People should be able to:
- See upcoming sessions
- Register
- Watch past replays
- Browse by theme (Capital / Planetary Party / Sanctuaries)

Sessions should be:
- Streamable live to public platforms (YouTube, etc.)
- Archived afterward
- Repurposed into clips, newsletters, and posts

## 5. Scalability & Moderation

Must support:
- Prevention of Zoombomb-style disruptions
- Handling 30-100+ attendees without stress
- Mute all by default
- "Raise hand" or queueing to speak
- Granting and revoking speaking permissions
- Assigning moderators if it grows

## 6. Technical Support Request Summary

Looking for support to:
- Design a lightweight, scalable tech stack that meets the above needs
- Set up: registration forms, speaker intake forms, live room configuration, backend database, public session hub
- Automate: registration confirmations, room access links, publishing workflows
- Build in: moderation controls, anti-disruption safeguards, growth-ready architecture
- Brief training on: admitting participants, mute/unmute, managing speakers, locking rooms, publishing replays
