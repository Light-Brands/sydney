**For Administrator Use — Hiring & Coordination Guide**

This outlines the different types of support needed to launch and sustain Live with Syd as a values-aligned, scalable, and operationally sane ecosystem.

Each role includes:

- Core responsibilities
- Skill profile
- Time horizon (setup vs. ongoing)
- Remote vs. in-person needs
- Where roles can be combined
- What we ask candidates to submit (scope + budget)

---

## Category 1 — Tech Infrastructure & Systems

### 1.1 Tech Architect / Systems Integrator

**Purpose:**
Design and implement the full tech stack in alignment with privacy, ethics, and scalability constraints.

**Core Responsibilities:**
- Finalize tool stack (Jitsi / BigBlueButton / Tally / Airtable / Notion / OBS)
- Configure live room defaults (mute, lobby, permissions)
- Build registration and speaker intake flows
- Connect automations (email, session reminders, approvals)
- Set up backend databases
- Embed public views into Notion
- Document system architecture
- Train Syd on core workflows

**Skill Profile:**
- No-code / low-code systems
- Airtable + automation tools (Zapier / Make / Peltarion)
- Video conferencing platform setup
- Privacy-first infrastructure
- UX simplicity mindset

**Time Horizon:**
- Heavy lift: 2-4 weeks
- Light maintenance: as-needed

**Remote vs In-Person:**
- 90% remote
- Optional in-person day for studio/tech setup (see Role 1.3)

**Can Be Combined With:**
- Automation Engineer
- Streaming Producer (early stage)

**Ask Candidates to Submit:**
- Fixed-price or hourly estimate
- Timeline
- Tool stack rationale
- Ongoing maintenance cost (optional retainer)

---

### 1.2 Automation Engineer

**Purpose:**
Make the system run without manual bottlenecks.

**Core Responsibilities:**
- Build Tally -> Airtable -> Email automations
- Configure reminder flows (24h + 1h)
- Set up post-session replay emails
- Toggle between auto-approve and manual approval
- Create admin dashboards
- Monitor error handling

**Skill Profile:**
- Zapier / Make / Peltarion
- Email automation (ConvertKit / Buttondown / Ghost)
- Data hygiene
- Edge-case testing

**Time Horizon:**
- Setup: 1-2 weeks
- Ongoing: 2-5 hrs/month

**Remote vs In-Person:**
- Fully remote

**Can Be Combined With:**
- Tech Architect (early stage)

**Ask Candidates to Submit:**
- Setup cost
- Monthly maintenance cost
- Automation map
- Error response protocol

---

### 1.3 Studio / On-Site Tech Setup Specialist

**Purpose:**
Ensure the physical hosting environment is professional, stable, and low-stress.

**Core Responsibilities:**
- Design filming setup (camera, mic, lighting, backdrop)
- Configure OBS scenes
- Optimize audio quality
- Install and test all software
- Build a "go live in 1-click" workflow
- Stress-test bandwidth and backups
- Create a physical studio SOP

**Skill Profile:**
- AV production
- OBS
- Live streaming
- Home studio design
- Troubleshooting under pressure

**Time Horizon:**
- One-time setup: 1-3 days
- Optional quarterly tune-ups

**Remote vs In-Person:**
- Ideally in-person
- Remote possible if Syd is tech-comfortable

**Can Be Combined With:**
- Streaming Producer

**Ask Candidates to Submit:**
- Day rate
- Equipment recommendations
- Setup timeline
- Ongoing support cost (optional)

- Calm under pressure

## Role Consolidation Map

For early-stage simplicity:

### Minimum Viable Team (4-6 people)

| Combined Role | Covers |
|--------------|--------|
| Tech Architect + Automation | Full stack build |
| Live Moderator + Community Manager | Live + Discord |
| Content Producer + Streaming | Media layer |
| Guest Liaison + Program Curator | Programming |
| Heart Sharing Facilitator(s) | Sacred container |
| Administrator | Operations |

### As You Scale

- Split each combo into specialized roles
- Add backup moderators
- Add bioregional coordinators

---

## What to Ask Every Candidate

The administrator should request:

1. Role(s) they want to take on
2. Weekly hours capacity
3. Monthly or project-based funding ask
4. Timeline for setup
5. Tools they're fluent in
6. Alignment statement (why this work matters to them)
7. Boundaries (what they will not do)
8. Growth vision (how their role could evolve)
